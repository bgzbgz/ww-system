# Implementation Plan: Boss Office Upload Command

**Branch**: `006-boss-office-upload` | **Date**: 2026-01-24 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/specs/006-boss-office-upload/spec.md`

---

## Summary

Implement file upload functionality for Boss Office where Boss can drag-and-drop or select a file (PDF, DOCX, TXT, MD) to create a Job entity in DRAFT state. Files are validated client-side (type, size) with immediate feedback, stored for Factory transmission, and associated with a Job record in MongoDB.

---

## Technical Context

**Language/Version**: TypeScript 5.x / HTML5 / CSS3
**Primary Dependencies**: HTML5 File API, Drag and Drop API, Fast Track CSS (spec-005)
**Storage**: MongoDB (jobs collection), File storage service (backend)
**Testing**: Jest (unit), Manual browser testing
**Target Platform**: Desktop browsers (Chrome, Firefox, Edge, Safari)
**Project Type**: Web application (existing Boss Office frontend)
**Performance Goals**: File validation < 500ms, Job creation < 3 seconds
**Constraints**: Max file size 10MB, single file only, no content parsing
**Scale/Scope**: Single user (Boss), < 100 jobs/day expected

## Implementation Phases

### Phase 1: Frontend Upload Component (Priority: P1)

**Goal**: Working dropzone with file validation

**Tasks**:
1. Create upload CSS styles extending brand system
2. Create dropzone HTML component
3. Implement file validation logic (type, size, empty)
4. Implement drag-and-drop event handlers
5. Implement file picker integration
6. Add file info display (name, type badge, size)
7. Add error message display
8. Add remove/clear functionality

**Validation**: Manual test with all file types and error conditions

### Phase 2: Backend API (Priority: P1)

**Goal**: Job creation and file storage

**Tasks**:
1. Create Job mongoose model
2. Create file storage service
3. Implement POST /api/boss/jobs endpoint
4. Add file upload middleware
5. Add validation middleware
6. Implement job creation logic
7. Connect file storage to job

**Validation**: API tests with valid and invalid files

### Phase 3: Integration (Priority: P1)

**Goal**: End-to-end upload working

**Tasks**:
1. Connect frontend to API
2. Handle success response (show job_id)
3. Handle error responses
4. Add loading state during upload

**Validation**: Full E2E test of upload flow

### Phase 4: Polish (Priority: P2)

**Goal**: Complete brand compliance and edge cases

**Tasks**:
1. Verify all brand compliance (colors, borders, typography)
2. Test all edge cases (network error, storage error)
3. Add retry functionality for errors
4. Performance testing with 10MB files

**Validation**: Run requirements checklist
