# Implementation Plan: Factory Integration

**Branch**: `007-factory-integration` | **Date**: 2026-01-24 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/specs/007-factory-integration/spec.md`

---

## Summary

Implement Factory webhook integration for Boss Office. Boss can submit DRAFT jobs to the canonical Factory webhook, view submission status, and retry failed submissions. All jobs go to one endpoint; routing happens inside n8n, not Boss Office.

---

## Technical Context

**Language/Version**: TypeScript 5.x (extends spec-006 codebase)
**Primary Dependencies**: Native fetch API, AbortController, Express.js
**Storage**: MongoDB (existing job collection), File storage (spec-006)
**Testing**: Jest (unit), Manual testing
**Target Platform**: Desktop browsers + Node.js backend
**Project Type**: Web application (existing Boss Office)
**Performance Goals**: Submit response < 5 seconds (excluding network)
**Constraints**: 30 second timeout, single webhook only
**Scale/Scope**: Single user (Boss), < 100 submissions/day

## Implementation Phases

### Phase 1: Backend Model Extension (Priority: P1)

**Goal**: Job model supports FAILED_SEND status and new fields

**Tasks**:
1. Add FAILED_SEND to JobStatus enum
2. Add submitted_at, last_attempt_at, failure_reason fields
3. Update JobSchema for MongoDB
4. Update JobResponse interface

**Validation**: Unit tests for model changes

---

### Phase 2: Factory Service (Priority: P1)

**Goal**: Backend can send jobs to Factory webhook

**Tasks**:
1. Create factory.ts service with LOCKED webhook URL
2. Implement buildWebhookPayload() function
3. Implement submitJobToFactory() with fetch + AbortController
4. Handle success (200), error (non-200), timeout
5. Return structured result for route handling

**Validation**: Unit tests with mocked fetch

---

### Phase 3: Submit Route (Priority: P1)

**Goal**: API endpoint for job submission

**Tasks**:
1. Add POST /api/boss/jobs/:jobId/submit endpoint
2. Validate job exists and status is submittable
3. Call factory.submitJobToFactory()
4. Update job status based on result
5. Return appropriate response

**Validation**: API tests with mock Factory

---

### Phase 4: Frontend Submit UI (Priority: P1)

**Goal**: Boss can submit and retry jobs

**Tasks**:
1. Create factory.ts API client
2. Create job-status component with badges
3. Add "Submit to Factory" button for DRAFT jobs
4. Add "Retry" button for FAILED_SEND jobs
5. Implement loading state during submission
6. Show success/error messages

**Validation**: Manual browser testing

---

### Phase 5: Integration & Polish (Priority: P2)

**Goal**: Complete end-to-end flow

**Tasks**:
1. Wire frontend to backend
2. Test full submit flow
3. Test full retry flow
4. Verify brand compliance (colors, typography)
5. Verify all error messages

**Validation**: Run requirements checklist
